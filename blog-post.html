Building an AI Chatbot with TF-IDF and Gemini

📅 March 2025 ⏱ 5 min read 👤 Rajat Kumar
Python TF-IDF Gemini AI NLP

When I set out to build an AI chatbot for a sports store, I wanted something that could handle quick product questions and complex, nuanced queries. The solution? A hybrid approach combining TF-IDF for speed and Google's Gemini AI for intelligence.

The Problem

Sports stores deal with hundreds of products, prices, offers, and services. Customers want instant answers. A simple keyword-matching bot fails on nuanced questions, but a full LLM call for every query is slow and expensive.

The Hybrid Solution

I designed a two-tier system:

  1. TF-IDF Layer — Handles simple, direct questions by matching keywords against a product database. Fast, reliable, and doesn't require API calls.
  2. Gemini AI Layer — Kicks in when TF-IDF confidence is low. Handles ambiguous queries, comparisons, and recommendations.

Key Implementation Details

TF-IDF Vectorization

I used scikit-learn's TfidfVectorizer to convert product descriptions and FAQs into numerical vectors. When a user asks a question, it's vectorized and compared against the knowledge base using cosine similarity. 

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(knowledge_base)

def get_best_match(query):
    query_vec = vectorizer.transform([query])
    similarities = cosine_similarity(query_vec, tfidf_matrix)
    best_idx = similarities.argmax()
    return knowledge_base[best_idx], similarities[0][best_idx]

Context Memory

One of the most important features was conversation memory. The chatbot remembers the user's last topic so follow-up questions like "What's the price?" work without the user repeating the product name.

Building a chatbot isn't just about answering questions — it's about understanding context and maintaining a natural conversation flow.

Results

The hybrid approach reduced API calls by ~70% while maintaining high-quality responses. Simple product queries are answered in under 100ms, and complex queries get intelligent responses from Gemini AI.

Lessons Learned

  • Start simple — TF-IDF handles more than you'd expect
  • Always add conversation context/memory
  • Set confidence thresholds to decide when to escalate to LLM
  • Cache frequent queries for better performance
